Al ejecutar el comando "node server.js", el entorno de Node.js inicializa el script, el framework Express configura las rutas definidas y el proceso se queda "escuchando" de forma asíncrona en el puerto especificado. La terminal se bloquea temporalmente para mantener el proceso del servidor vivo y receptivo a peticiones externas.

En la terminal aparece el siguiente mensaje:
"Servidor ejecutándose en puerto 3000"

Cuando se visita la ruta "/estado" desde el navegador o un cliente HTTP, el servidor procesa la solicitud GET, localiza el endpoint correspondiente y responde enviando un objeto en formato JSON con el estado actual del servicio. El navegador renderiza el siguiente resultado en pantalla:

{
  "estado": "Servidor funcionando",
  "servicio": "API comunitaria"
}
